# 风险分析

基于信号图的**双维度风险评分**：功能逻辑复杂度 × 时序复杂度。

## 算法原理

### 节点分类

每个节点首先被分为三类：

| 类别 | 识别方式 |
|------|----------|
| ⏰ 时钟 | `is_clock=True` 或名称含 `clk` |
| 🔄 复位 | `is_reset=True` 或名称含 `rst` |
| 📊 数据 | 其他信号 |

时钟和复位独立分析，不混入评分。

### 双维度评分 (实际实现)

```
功能逻辑复杂度 = fan_in×3 + fan_out×2 + 15(汇聚) + 10(发散)

时序复杂度     = 15(寄存器) + fan_in×2
```

**阈值**: CRITICAL≥40, HIGH≥25, MEDIUM≥15, LOW<15

> 📝 注：实际实现 (signal_graph_viewer.py:158-188) 使用简化公式。
> 文档中的 width×0.3、12(无SVA)、8(无Cov)、pipeline_depth×5 等为规划中功能。

## CLI 用法

```bash
# 文本输出
python run_cli.py risk analyze -f top.sv

# JSON 输出
python run_cli.py risk analyze -f top.sv --json

# 配置组合逻辑深度阈值（默认 3）
python run_cli.py risk analyze -f top.sv --max-comb-depth 5
```

## 关键指标说明

| 指标 | 说明 |
|------|------|
| fan_in | 驱动该信号的节点数。≥3 表示多路汇聚，优先级/仲裁逻辑复杂 |
| fan_out | 该信号驱动的节点数。≥3 表示广播信号，扇出负担重 |
| pipeline_depth | 从主输入到该节点经过的寄存器级数 |
| is_conv | fan_in ≥ 3，汇聚节点 |
| is_div | fan_out ≥ 3，发散节点 |

## 应用场景

1. **验证覆盖缺口识别**：高风险但无 SVA/Coverage 的信号需要补充验证
2. **设计复杂度评估**：高 fan-in/fan-out 的信号需要特别关注
3. **时序瓶颈定位**：pipeline_depth 深的信号可能是关键路径
