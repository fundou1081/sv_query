## A 类: 代码 Bug (可修复)

### Issue 28: 注释混入节点名

**根因位置**: `graph_builder.py` → 信号名提取逻辑

**根因分析**:
```python
# 当前行为
信号名提取时，注释被当作名称的一部分
例如: "// registering input of the multiplier a_int"
被解析为节点名的一部分

预期行为
注释应该被过滤，只保留 "a_int"
```

**直接原因**: `_get_signal()` 或节点创建时未过滤注释

**修复方案**:
1. 在 `clean_name()` 或节点创建时过滤注释
2. 正则过滤: `//.*$` 和 `/\*.*\*/`

---

### Issue 33: 字面量作为节点名

**根因位置**: `graph_builder.py` → `_get_all_signals()` 或边创建逻辑

**根因分析**:
```python
# 当前行为
字面量 "0", "1'b0" 被创建为节点
边: serv_top.0 -> serv_top.iscomp (DRIVER)

预期行为
字面量不应该创建节点，只创建边
边: 1'b0 -> serv_top.iscomp (DRIVER)  # 无源节点
```

**直接原因**: `_get_all_signals()` 返回了字面量，并在后续创建了节点

**修复方案**:
1. 在 `_get_all_signals()` 中检测字面量并跳过
2. 或在节点创建时检测字面量并跳过

---

### Issue 34: 循环变量出现在节点名

**根因位置**: `graph_builder.py` → 数组索引处理

**根因分析**:
```python
# 当前行为
gpu.lsu_read_valid[lsu_index]  # lsu_index 是循环变量
gpu.core_lsu_read_valid[j]    # j 是循环变量

预期行为
应该是具体的循环索引值，或者使用通用表示
gpu.lsu_read_valid[0-3]  # 或 gpu.lsu_read_valid[i]
```

**直接原因**: 循环变量的 `for (genvar i=0; i<N; i++)` 未被解析/展开

**修复方案**:
1. 解析 for 循环的边界，展开为具体索引
2. 或使用变量名 `[i]` 替代实际索引

## D 类: 功能缺失 (需新增功能)

### Issue 21: 参数表达式未展开 (ADDR_WIDTH-1)

**根因位置**: `base.py` 或 `graph_builder.py` → 参数应用逻辑

**根因分析**:
```
1. 参数 ADDR_WIDTH=3 被解析
2. 但节点名仍为 "in[ADDR_WIDTH-1]" 而非 "in[2]"
3. 参数值未被应用到节点名中
```

**直接原因**: 节点创建时使用原始信号名，未替换参数

**修复方案**:
1. 在节点创建前，解析参数表达式
2. 替换节点名中的参数引用

### Issue 29: 复杂参数未展开 (CVA6Cfg)

### Issue 31: 位宽(0,0)

**根因位置**: `base.py` → `extract_port_width()`

**根因分析**:
```
cva6.clk_i PORT_IN (0,0)
预期: clk 是 1-bit，应该是 (0,0) 或 (1,0)
实际: (0,0) 表示什么？

分析:
- (1,0) = 1-bit (msb=1, lsb=0)
- (0,0) = 未识别？还是 1-bit？
```

**直接原因**: 端口位宽解析失败，返回默认值 (0,0)

**修复方案**:
1. 确定 (0,0) 的含义
2. 如果是未识别，改进解析逻辑
3. 如果是 1-bit，修改默认值

## 解析范围问题根因

### 问题本质

sv_query 当前的行为：只解析传入的文件，不会自动发现同目录或子目录的相关文件。

### 影响

1. **边数为0或很少**: 子模块未解析，无法建立连接边
2. **实例连接数为0**: 端口定义未知，连接无法建立
3. **LOAD边为0**: 端口方向未知，不创建边

### 解决方案讨论

| 方案 | 描述 | 优缺点 |
|------|------|--------|
| 方案A | 改进解析范围自动发现 (递归扫描) | 优点：自动；缺点：可能解析不需要的文件 |
| 方案B | 要求用户显式传入所有文件 | 优点：可控；缺点：不方便 |
| 方案C | 智能推断：解析同目录所有.v/.sv文件 | 优点：平衡；缺点：可能有重复 |
