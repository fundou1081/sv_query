
## 18:50 pyslang SIGSEGV 问题调查

### 问题现象
- 单独运行各测试目录全部通过 (regression 272, unit 30, integration 193)
- 一起运行时出现 SIGSEGV (段错误)
- 问题与测试加载顺序有关：test_interface_instance → test_modport_direction 会崩溃

### 已排除的原因
1. ✅ 单独 import 两个模块没问题
2. ✅ 单独 loadTestsFromModule 没问题
3. ✅ 单独运行 suite 没问题
4. ❌ 但 combine 两个 suites 然后运行会导致 SIGSEGV

### 发现的问题模式
Combine suites 后运行时崩溃，而不是在 combine 时崩溃。
这说明问题与 TestSuite 的内部状态管理有关，可能与 pyslang AST 节点引用有关。

### workaround
目前所有测试目录单独运行都正常。可以通过 subprocess 隔离运行避免这个问题。

### 根本原因推测
pyslang 的 AST 节点可能是用 C/C++ 实现的，节点之间有复杂的引用关系。
当 unittest.TestSuite 管理多个测试模块的测试生命周期时，这些引用可能被不当释放。

### 已采用方案
使用 subprocess 隔离运行各测试目录，避免 pyslang 内存问题。

## 01:18 Phase 3 推进 - IdentifierNameSyntax 修复

### 问题根因
`inside {[1:10]}` 约束的变量提取失败，因为 `IdentifierNameSyntax` 没有 `name` 属性，只有 `identifier` 属性。

```python
# 错误
name = str(getattr(expr, 'name', '') or ...)

# 正确
ident = getattr(expr, 'identifier', None)
if ident and hasattr(ident, 'value'):
    name = str(ident.value).strip()
```

### 代码改动
- `constraint_visitor.py` `_extract_vars_from_expr()`: Identifier 分支增加 `identifier` 属性检查
