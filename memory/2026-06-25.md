  - Pass 1: 替换 top prefix (`ariane.* → cva6.*`)

- CVA6 自动剥 4 个 wrappers: `i_cva6`, `impl`, `i_rr_arb_tree`, `genblk1`

- 现象: extract_module 返回的 inst IDs 含 wrapper segment (`darksocv.bridge0.core0`)
       namespace rewrite 只改 MIG port_to_internal, 没改 inst IDs
       当 wrapper candidate 跟真实 inst 同名时 → edges=0 (silent!)
- 修复: extract_module 返回的 inst IDs 也用同一套 rewrite_path 改写
- 用 `dataclasses.replace(inst, id=new_id)` 同步 rewrite

## 🔮 pyslang Native API 发现 (2026-06-25)

### 完整 API 列表 (pyslang 11.0.0)

```python
import pyslang

root = compilation.getRoot()
for top in root.topInstances:
    inst.hierarchicalPath   # 完整路径 "ariane.i_cva6.i_frontend"
    inst.lexicalPath        # 词法路径
    inst.arrayPath          # array index
    inst.body               # InstanceBodySymbol (iterable children)
    inst.definition         # module 定义 symbol
    inst.parentScope        # 父 scope
    inst.portConnections    # 端口连接列表 (native, 不需 sv_query build MIG)
    inst.visit(callback)    # visitor pattern
```

### 性能对比 (CVA6, 73 instances, depth 10)

| 方法 | 时间 | Speedup |
|------|------|---------|
| sv_query 当前 (semantic_adapter.get_module_instances) | ~265ms | 1.0x |
| pyslang native (topInstances + body iterator) | ~60ms | **4.4x** |

### Native 优势
1. `.hierarchicalPath` 自动算好 (不需要 sv_query 拼 path)
2. `.portConnections` 直接给 (不需要 sv_query build MIG)
3. `.visit(callback)` visitor pattern (single-pass traversal)
4. `.body` InstanceBodySymbol 干净 children iterator

### Native 限制
1. `topInstances` 含所有 utility cells (24 个), 需 filter by user target
2. `.hierarchicalPath` 用 pyslang 第一个 top instance 名字 (`ariane`) 而不是 user-specified (`cva6`)
   - 这正是 namespace rewrite 要解决的问题
   - **改用 native API 可能完全消除 namespace rewrite 的需求**
