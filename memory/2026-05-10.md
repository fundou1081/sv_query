## 17:32 - Generate 块支持继续

### 调研结果
pyslang 解析 generate for 循环时：
- `LoopGenerateSyntax.block` 返回 `GenerateBlockSyntax`
- 当迭代 `block` 时，得到的是结构化内容（tokens 等），而非 `block.members` 里的实际内容
- `block.members[0]` 才是 `HierarchyInstantiationSyntax`

### 原因分析
GenerateBlockSyntax 实现了 `__iter__`，迭代它得到的是语法结构，而非 members 属性指向的实际内容。

### 已实现方案
在 `base.py` 中添加了 `get_generate_instances()` 方法：
```python
def get_generate_instances(self, trees: dict = None) -> List:
    # 遍历找 LoopGenerate
    # 从 LoopGenerate.block.members 直接提取 HierarchyInstantiation
```

ConnectionExtractor 处理 generate 实例时需要特殊处理，因为：
- generate 实例的 `inst.parent` 不是 `ModuleDeclaration`
- 需要修改 ConnectionExtractor 来处理这种情况
